# ClawLike 🦞

ClawLike is a lean, modular autonomous agent framework designed for the solo power-user. It bridges the gap between precision pair-programming and remote autonomous automation.

## 🏗️ The Construction Philosophy
This project is built using a unique **Multi-Agent Orchestration** workflow. We prioritize high-level architectural integrity and "unbloated" logic.

### The Team
- **The Architect (Gemini)**: Responsible for design patterns, structural safety, and the long-term roadmap.
- **The Builder (Claude)**: Responsible for implementation, logic, testing, and high-speed construction.
- **The Human Engineer**: The pilot who orchestrates the hand-offs and provides real-world verification.

## 🔄 Development Protocol
We follow a strict "Bridge" protocol to ensure the Architect and the Builder stay aligned:

1. **Design**: The Architect defines a task in the `CLAW_CONSTRUCTION_PROTOCOL.md`.
2. **Build**: The Builder reads the protocol and the `SPEC_CLAWLIKE_BRIDGE.md` to implement the code.
3. **Log**: Every change is recorded in the `DEVELOPMENT_LOG.md`.
4. **Verify**: The Architect reviews the construction and plans the next phase.

### 📁 Temporary Synchronization Files
These files serve as the live "Bridge" between agents and will be removed upon project completion:
- `SOUL.md`: Agent Trinity Identities, Boundaries, and Loop Cycle.
- `SPEC_CLAWLIKE_BRIDGE.md`: Technical Specification, Structure, and Decision Log.
- `CLAW_CONSTRUCTION_PROTOCOL.md`: Active Task Handoff & Sync Protocol.
- `IMPLEMENTATION_PLAN.md`: Strategic Roadmap.
- `DEVELOPMENT_LOG.md`: Real-time Construction Journal.

## 🛡️ Principles
- **Clarity Over Bloat**: No unnecessary dependencies.
- **Security First**: All system-wide actions require explicit human approval.
- **State-Aware**: Seamlessly switch between local assistance and remote autonomy.
